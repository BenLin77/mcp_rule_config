description: 單獨呼叫的 Spec Agent Workflow
---

```yaml
workflow:
  name: Spec Agent Workflow
  steps:
    - name: Spec-Agent
      run: |
        /spec-agent "你是系統分析師與規格設計者。請在 *不要要求使用者先輸入長篇需求描述* 的前提下，自動從目前 Workspace 的程式碼推導出系統行為，並產出一份結構化的規格（可直接放入 Spec Kit），內容包含：

        【自動掃描行為】
        1) 探索專案結構：找出主要入口（API、CLI、batch scripts）與核心業務模組。
        2) 從函式與類別的命名、參數與 docstring，推導：
           - 功能目標與典型使用情境。
           - 輸入輸出資料結構與主要處理步驟。

        【規格內容】
        請根據實際程式碼整理出：
        1) 功能目標與使用情境。
        2) 資料流與主要處理步驟（可用流程條列或簡圖描述）。
        3) 介面定義：
           - 重要函式/方法的參數與回傳值。
           - CLI 參數、環境變數、設定檔 key 等。
        4) 錯誤情境與邊界條件：常見錯誤來源、例外處理策略。
        5) 效能與安全性要求：從現有實作中推測出的假設與限制。

        請用清楚分段與小標題輸出，方便我直接存成 spec 檔案，並在可能的情況下附上檔名/模組名稱以方便對照。" 
```
